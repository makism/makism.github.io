mtggg & mtghello

As part of shaperning my Python/Spark/etc. skills, I started a project for analyzing MTG cards. The whole project is separated into multiple modules. The main project is hosted in mtggg and some nice visualizations and results will be available on mtghello.


Data Analysis

The cards are given in a standard JSON format and they are fed to Apache Spark. Preprocessing is the next natural step. Duplicate cards are removed, strings and categories are encoded, MTG keywords are extracted from the text, etc.. All the analysis takes place in JupyterLab. The results are written both in Apache Parquet format, and they are dumped into both to MongoDB and Elasticsearch for later use.

Search Engine - Recommendation Engine

Built upon the previous layer, a simple interface is built to query ElasticSearch. Regarding the recommendation engine; the purpose is to explore the construction of a card recommendation system using different features; for now, I am experimenting with a UMAP model.

REST API

The idea is to use Flask to serve the machine learning models generated from the previous layers of the stack.

Automation

The whole analysis is hosted in a Vagrant box for convinience; some operations, such as fetching the raw cards’ metadata is done through Airflow.
