Projects

Below, I would like to list some of my most important and close to my heart projects, that I have spearheaded or just participated. Some of these projects are still in active development, while others are presented because I feel nostalgic about them.

dyfunconn: Dynamic Functional Connectivity - A NeuroImaging Python Module

dyfunconn is a Python module for your EEG/fMRI/and MEG-i-guess connectivity and graph analysis needs. It is organized into a few sub-modules; the most obvious ones are the fc for functional connectivity and graph for the graph-related algos: graph distances and thresholding schemes. There some exciting stuff going on the ts sub-module also: Markov Matrices, Entropy, Wald Test, to name a few. I followed and employed the “guidelines” set from other major scientific projects (i.e., numpy, scikit-learn, etc.) regarding the API specification, the documentation, continuous integration and code coverage, and testing.

Big data and parallel processing pipeline (real title: unnamed)

My proposal focuses on separating the client (as in a network client) from the rest of the (internal) components. A quick overview of the pipeline is as follows. A client application can submit several tasks (with the order of the execution being a possibility) in a JSON format to a simple RESTful application developed in Flask. The uploaded file is then parsed and converted into a DAG and submitted to Airflow. Apache Airflow is configured to run in a cluster-mode, using Celery and RabbitMQ; distributing and executing the jobs across the cluster’s nodes.

There an initial, demo application and setup of this proposal.

Future developments include training a machine learning model with each job’s settings (arguments, input files, metadata, and execution time). The model could later provide an estimation of the execution time of future jobs with similar settings. Another interesting feature would be to provide real-time information about the jobs’ state using a streaming feed (using RabbitMQ or Kafka).

EEG/fMRI in Real-Time

Part of a small team; we developed a GUI application (in C++/Qt) to stream, visualize, and process EEG/fMRI signals simultaneously in real-time. The challenge arises because of the very high sampling rate of 5000Hz. We had to make sure that all these operations complete within a pre-defined time window. It was a unique experience that I genuinely enjoyed contributing to all the different aspects and phases of the project. I implemented algos for motion correction and filtering, developed tools for visualization, the plugin interface, network backend, performed real-world testing, etc.

A feature-complete PHP5 MVC framework

I jumped onto the-then hype train of MVC frameworks from the very beginning. I was fascinated by the elegance that these frameworks brought into the messy world of scripts; and the sophisticated design patterns. Decided to educate me on the topic, first and foremost by reading the “Design Patterns: Elements of Reusable Object-Oriented Software” by the “Gang of Four” and hacking around and tearing down the code of other frameworks. The final result was leaf framework. It contained all the usual features one would expect to find to an MVC framework: URL routing, scaffolding, a simple ORM, hooks, plugins, multiple locales, etc. I even went the extra mile and profiled the code to identify the bottlenecks (spoiler: it was the various regular expression in the URL routing class). This spiraled into a new “minimal” version using static routes but extremely fast!
